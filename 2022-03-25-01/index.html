資料庫的概念

在上面我們成功的創建了一個我們的資料庫，在這裡我們將學習如何基本的存取以及修改資料庫

Database、Collection和Document

首先我們有一些重要觀念必須先了解，一個MongoDB中允許建立多個Database，而一個Database中可以存放多個Collections，最終在Collection中儲存一個一個的Document。通常我們會連結到某一個Database中，然後在每個Collection中儲存不同的資料。

Schema與Model

在MongoDB中，我們會使用model進行互動，而Schema就是定義在該集合(Collection)中Documents的架構，可以用來轉成Model，示例如下:

資料庫的基本操作

  • 新增
    • create
  • 查詢
    • findOne
    • find
  • 更新
    • updateOne
    • updateMany
    • findOneAndUpdate
  • 刪除
    • deleteOne
    • deleteMany
    • findOneAndDelete

另外這裡的操作都會需要用到Promise，可以利用async function去執行或是使用.then()

create

create() 用來插入Document，會回傳插入的該Document

findOne

會以 Object 型別回傳找到的第一個Document

find

跟上一個一樣，只是這個能搜尋好多個，但這個以 Array 回傳，裡面有所有符合搜尋條件的 Object

updateOne

顧名思義，就是更新依照條件找到的第一個Document，但不會回傳結果，如果要回傳結果，要用 findOneAndUpdate

updateMany

同上概念，但這次是更新依照條件找到的所有Document，也不會回傳結果

deleteOne

找到第一個符合條件的之後會刪除該Document，不會回傳結果，如果要回傳結果，要用 findOneAndDelete

deleteMany

基本上findOne與deleteOne相對於find與deleteMany，前者是第一個值，後者是全部

findOneAndUpdate

跟updateOne一樣，但會回傳 更新前的值

findOneAndDelete

跟deleteOne一樣，但會回傳值
